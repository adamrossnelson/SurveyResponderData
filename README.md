# SurveyResponderData

A static file host for non-confidential CSV and Parquet data files. This is **not** a software project — it is a companion data repository for [SurveyResponder](https://github.com/adamrossnelson/SurveyResponder) and [SurveyResponderMemos](https://github.com/adamrossnelson/SurveyResponderMemos).

## Why This Repo Exists

Large data files create bloat when stored alongside application code. By hosting data here in its own repository, we keep [SurveyResponder](https://github.com/adamrossnelson/SurveyResponder) and [SurveyResponderMemos](https://github.com/adamrossnelson/SurveyResponderMemos) lean while providing stable, publicly accessible URLs that anyone can use to load data directly from code.

**You never need to clone, pull, or fetch this repository.** All file management happens through the GitHub web interface. All data consumption happens via raw URLs.

```
|   ├── PRCA_LLM_AgreementScale_nan_audit.csv ← C) CSV file documenting NaN locations and treatements
|   ├── PRCA_LLM_AgreementScale_notes.txt     ← D) Optional txt file for notes regarding these data
│   ├── PRCA_LLM_AgreementScale_params.json   ← E) JSON from responder.run_write("PRCA_LLM_AgreementScale.csv")
│   ├── PRCA_LLM_FrequencyScale.csv           ← A) CSV from responder.run_write("PRCA_LLM_FrequencyScale.csv")
|   ├── PRCA_LLM_FrequencyScale_ready.csv     ← B) CSV ready for analysis (numeric and referse coded)
|   ├── PRCA_LLM_FrequencyScale_nan_audit.csv ← C) CSV file documenting NaN locations and treatements
|   ├── PRCA_LLM_FrequencyScale_notes.txt     ← D) Optional txt file for notes regarding these data
│   ├── PRCA_LLM_FrequencyScale_params.json   ← E) JSON from responder.run_write("PRCA_LLM_FrequencyScale.csv")
│   ├── MFQE_LlamaResponses.csv               ← A) CSV from responder.run_write("MFQE_LlamaResponses.csv")
|   ├── MFQE_LlamaResponses_ready.csv         ← B) CSV ready for analysis (numeric and referse coded)
|   ├── MFQE_LlamaResponses_nan_audit.csv     ← C) CSV file documenting NaN locations and treatements
|   ├── MFQE_LlamaResponses_notes.txt         ← D) Optional txt file for notes regarding these data
│   └── MFQE_LlamaResponses_params.json       ← E) JSON from responder.run_write("MFQE_LlamaResponses.csv")
└── DataIndex.md                              ← Index of available datasets
```

- Where `A) CSV from responder.run_write("{base_file_name}.csv")` is the raw csv file generated by the `run_write` method of the `SurveyResponder` class.
- Where `B) CSV ready for analysis (numeric and referse coded)` is the csv file that has been converted to numeric and appropriately reversed coded by `code tbd`.
- Where `C) CSV file documenting NaN locations and treatements` is the csv file generated by `code tbd` that documents NaN locations and treatements which explain how each NaN was handled.
- Where `D) Optional txt file for notes regarding these data` is an optional text file we can create to document thoughts, ideas, observations, of other exceptions related to that dataset.
- Where `E) JSON from responder.run_write("{base_file_name}.csv")` is the json file generated by the `run_write` method of the `SurveyResponder` class which contains reproducibility snapshot meta information of the run including `questions_path` — path to the questions text file; `persona_path` — path to the persona JSON file; `model_name` — Ollama model name being used; `base_url` — Ollama API endpoint; `num_responses` — number of synthetic respondents requested; `temperature` — generation temperature; `response_options` — allowed response choices; `run_date` — timestamp for when the run happened; `num_questions` — count of loaded questions; `questions` — the actual list of questions loaded from the file; `persona_dictionary` — the full persona dictionary loaded from the persona JSON; `computer_memory` — total system RAM in GB; `computer_os` — operating system and version string; `computer_python` — Python version string.

## How to Access the Data

Every file in the `data/` directory is available at:

```
https://raw.githubusercontent.com/adamrossnelson/SurveyResponderData/main/data/<filename>
```

### Python (pandas)

```python
import pandas as pd

# CSV
df = pd.read_csv(
    "https://raw.githubusercontent.com/adamrossnelson/SurveyResponderData/main/data/survey_results.csv"
)

# Parquet (preferred — smaller, faster, typed columns)
df = pd.read_parquet(
    "https://raw.githubusercontent.com/adamrossnelson/SurveyResponderData/main/data/survey_results.parquet"
)
```

### Python (Polars)

```python
import polars as pl

df = pl.read_csv(
    "https://raw.githubusercontent.com/adamrossnelson/SurveyResponderData/main/data/survey_results.csv"
)
```

### R

```r
library(readr)

df <- read_csv(
    "https://raw.githubusercontent.com/adamrossnelson/SurveyResponderData/main/data/survey_results.csv"
)
```

## Caching Warning

> **Note:** `raw.githubusercontent.com` is CDN-cached. After a file is updated, the old version may continue to be served for several minutes. For the workflow associated with this project the caching will not likely be problematic because the intention is to never change a csv file once it has been committed to the repository. However, important to not if you recently modified a csv the caching mechanism may serve the old version for several minutes.

## File Format Strategy

| File pattern (same base name) | Role in the dataset bundle | How it is created | Notes |
|---|---|---|---|
| `{base}.csv` | A) Raw responses export | `responder.run_write("{base}.csv")` | Primary raw output for a run |
| `{base}_ready.csv` | B) Analysis-ready data | Post-processing (`code tbd`) | Numeric + reverse-coded where appropriate |
| `{base}_nan_audit.csv` | C) Missing-data audit | Audit step (`code tbd`) | Documents NaN locations and treatments |
| `{base}_notes.txt` | D) Human notes (optional) | Manual | Observations/exceptions for the dataset |
| `{base}_params.json` | E) Reproducibility snapshot | `responder.run_write("{base}.csv")` | Run parameters + environment metadata used to generate `{base}.csv` |

When multiple files exist for the same dataset, the so-called "base name" (the `{base}` portion) is what associates the multiple files within that bundle together.

## File Naming Conventions + REminders

Use lowercase with underscores — no spaces, no capital letters.

Each dataset is identified by a unique base name** (`{base}`). The `{base}` portion must be unique within `data/`, because it is what associates all companion files that belong to the same dataset bundle.

For every raw dataset export, there will be an accompanying parameter snapshot JSON file named `{base}_params.json`.

Optionally, you may also include a notes file named `{base}_notes.txt`.

To keep these paired, use the same base name for all companion files:

```
data/
├── res_qs_25.csv                ← Exported by `run_write` method
├── res_qs_25.json               ← Preserved by `run_write` method
├── res_qs_25_ready.csv          ← Analysis-ready data
├── res_qs_25_nan_audit.csv      ← Missing-data audit
├── res_qs_25_notes.txt          ← Human notes (optional)
├── survey_res_2025_03.csv       ← Exported by `run_write` method
├── survey_res_2025_03.json      ← Preserved by `run_write` method
└── ...
```

This naming convention makes it immediately clear which JSON belongs to which CSV. When browsing the `data/` directory, paired files will sort next to each other alphabetically.

If you need distinct versions to coexist, use date-stamped filenames (e.g., `survey_results_2025_03.csv` / `survey_results_2025_03.json`).

## File Size Limits

- GitHub warns at files over 50 MB and blocks files over 100 MB.
- For any CSV larger than ~10 MB, prefer Parquet.
- Keep all individual files under 50 MB whenever possible.

## How to Contribute / Update Files

All uploads happen through the **GitHub web interface**. Do **not** clone this repository.

1. Navigate to the [`data/`](data/) folder on GitHub.
2. Click Add file → Upload files.
3. Drag and drop your `.csv` or `.json` or `.txt` file(s).
4. Write a brief commit message describing the change.
5. Commit directly to `main`.

Avoid revising a CSV file once it has been committed. 

For full contribution guidelines, see [CONTRIBUTING.md](CONTRIBUTING.md).
